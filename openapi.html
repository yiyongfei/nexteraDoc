主要步骤

生成密钥

点击“生成密钥”按钮，弹出生成密钥页面，在该页面填写密钥名称，如果该密钥永久有效就不用填失效时间，否则填上失效时间（这里失效时间是指零时，比如填2024年2月1日，那该密钥实际有效时间 是2024年1月31日24时）。

这里的API Key和API Secret用于获取访问凭证，请保存好，注意防止泄露。

可以通过“查看”链接对现存的密钥进行维护，或者再次查看Key和Secret。

点击“开发API”按钮，会打开接口SwaggerUI页面，可直接在此做接口验证。生成对话建议在Postman验证，会通过Stream流方式输出。

获取访问凭证

获取访问凭证有二种方式，任选其中一种即可。

请求地址：https://openapi.nextera.chat/oauth/token

请求头

无

请求参数

名称类型是否必填描述
apiKeyString必填密钥里的API Key
apiSecretString必填密钥里的API Secret
grantTypeString必填授权类型，可选[client_credentials|refresh_token]，新获取访问凭证用client_credentials，刷新凭证用refresh_token
refreshTokenString非必填refreshToken，来源于获取访问凭证接口调用结果，当grantType为refresh_token时必须提供

注意，grantType为refresh_token时，如果之前生成的访问凭证尚未过期，refresh_token会将之前的访问凭证注销掉，但grantType为client_credentials时不会注销之前的访问凭证。

响应说明

名称类型描述
codeNumber请求结果代码，20000表示成功，其它数值表示失败
messageString请求结果内容
exceptionCodeNumber异常编码
exceptionTraceString异常结果，当code非20000时，会体现异常情况
result对象请求结果
accessTokenString访问凭证，调用其他接口需要访问凭证。
refreshTokenString刷新用凭证，访问凭证过期后刷新用。
expiresInString凭证失效时间，最长10整天，最短密钥的失效时间。样例表示2024年1月30日零时过期。

请求地址：https://openapi.nextera.chat/api/v1/session/create

激活会话

长期未对话导致会话失效，此时需要重新激活会话才可继续对话，会话里的历史对话记录仍旧保留。如果该会话从未曾对话过，没有历史对话记录，会激活失败，需重新创建会话。

请求地址：https://openapi.nextera.chat/api/v1/session/activate

请求头

开始对话

用户的多轮对话，只需保持会话标识一致。所有对话都归属于账户，所以无论通过哪个访问凭证开启对话，这些对话都属于同一账户。外部系统需要自行将会话和使用用户进行关联。

请求地址：https://openapi.nextera.chat/api/v1/chat

请求头

名称类型是否必填描述
accessTokenString必填获取访问凭证接口返回的accessToken

请求参数

名称类型是否必填描述
chatRequest对象必填
sessionString必填会话标识
contentString必填对话内容
settingCodeString必填应用编码，查询应用的结果里获得
images字符串集合非必填多模态里的图像,集合,每个元素是图像的URL

请求参数样例：

{
  "session": "8554d9a8b72f4d31b848bccaed3b60cf",
  "content": "我想考科目一，你出个题吧",
  "settingCode": "2dcc727838dd18b408313e80b9cfe804",
  "images": []
}
